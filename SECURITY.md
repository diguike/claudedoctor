# Security Policy

## Supported versions

Security fixes are provided for the latest published minor release. Upgrade to the newest npm version before reporting an issue that may already be fixed.

## Reporting a vulnerability

Please use [GitHub's private vulnerability reporting flow](https://github.com/diguike/claudedoctor/security/advisories/new). Do not open a public issue for a vulnerability that could expose credentials, write outside the intended shell profile, escape the Web root, or compromise the published npm package.

Include the affected version, platform, reproduction steps, impact, and any proposed mitigation. Never attach real OAuth tokens, API keys, raw credential files, or an unredacted `claudedoctor env` dump.

You should receive an initial response within seven days. Confirmed issues will be coordinated privately until a fix and advisory are ready.

Account suspensions and policy appeals are not product security reports. Use [Anthropic's official appeal process](https://support.claude.com/en/articles/8241253-safeguards-warnings-and-appeals).
